# PMO Agent

第三者検証会社のPMO業務（チケット追跡・品質分析・報告・相談）をAIで支援するシステムのコンテキスト。設計の全体像は [docs/DESIGN.md](docs/DESIGN.md) を参照。

## Language

**案件**:
顧客 × 対象開発プロジェクト × 契約期間で定まる、検証・PMO支援業務の単位。チケット接続・分析・報告書・LLM選択はすべて案件に紐づく。
_Avoid_: プロジェクト、検証プロジェクト、エンゲージメント

**顧客**:
検証サービスを契約するクライアント企業。
_Avoid_: クライアント、カスタマー

**対象開発プロジェクト**:
顧客側で進行する開発プロジェクト。案件の検証対象であり、チケットの発生源。
_Avoid_: 開発案件

**停滞チケット**:
定義済みの停滞条件（一定日数更新なし・期限超過など、条件は案件ごとに設定）に該当したチケット。
_Avoid_: 放置チケット、滞留チケット

**督促**:
停滞チケットへの対応を促すフォローアクション。本システムではPMOへの通知として実現し、顧客のチケットシステムへの書き込みは行わない（読み取り専用）。
_Avoid_: 追い回し、催促

**ODC分類**:
チケットに付与する直交欠陥分類（欠陥タイプ・トリガー・検出アクティビティ等の軸）。出所（チケットフィールド由来 / LLM推定）と確定状態（レビュー待ち / 確定済み）を必ず持ち、LLM推定は人の確定を経て分析に使われる。
_Avoid_: 欠陥分類、バグ分類

**診断**:
案件の定量メトリクスとナレッジ（開発標準等）を入力に、LLMが生成する品質所見・リスク・提言。人のレビューを経て報告書の材料になる。
_Avoid_: 分析結果、アセスメント
