当たり前品質 棚卸し

PMO Agent ／ 画面・機能別「あるべきベースライン」と現状ギャップの一次評価

作成日: 2026-07-12 ステータス: レビュー用ドラフト 根拠: コード経路の実査 + 類似ツール調査

このドキュメントの位置づけ

「当たり前品質（狩野モデル: must-be quality）」＝満たされても満足は生まれないが、欠けると強い不満・怒りを生む要素。 その中身は作り手ではなく使い手の暗黙の期待で決まり、要求仕様に書かれないため作り手側から列挙すると必ず抜ける。 そこで本書は「私の主観」ではなく、①実際のコード経路（URL/テンプレート）の実査と ②世の中の類似ツールが標準搭載する機能の調査の2点から、画面ごとの当たり前を客観的に定義する。

○ 概ね充足（当たり前を満たす） △ 部分的・深さ不足／要現物確認 ✗ 欠落（当たり前が未達）
現状評価はコードのURL経路・テンプレートからの一次評価です。挙動の深さ（実際に詰まるか）は、 次段の「通し点検（探索的テスト）」で最終確認します。当たり前品質は作り手側の机上評価では取りこぼすため、 この二段構えを前提とします。

★総括: 優先度つきギャップ一覧

当たり前品質の欠落を、ユーザーが実使用で踏み抜く痛みの大きさ順に並べたもの。

  1. チケットを開けない／作れない／編集できない
    一覧はあるが、行クリックは外部JIRA/Redmineへ飛ぶだけ。アプリ内に詳細・新規作成・編集・ステータス変更・コメントが一切ない。課題管理ツールの中核が空。既に本人が「致命的」と指摘済み。
  2. 計画（ガント）が閲覧専用
    タスクの追加・編集・削除・並べ替え・依存関係の設定・ドラッグでの日程変更ができない。計画ツールとして「編集できない計画表」は当たり前を満たさない。
  3. チケット一覧の絞り込みが貧弱
    検索が件名部分一致のみ。担当者・優先度・ステータス・期間での絞り込み、列ソート、一括操作、保存ビューがない。エンタープライズのデータテーブル標準に未達。
  4. ナレッジに詳細閲覧がない
    アップロード・削除・再インデックスはあるが、文書を開いて中身・チャンク・引用元を確認する画面がない。
  5. 各登録フォームの標準項目の充足確認
    リスク・テストケース等が、PMBOK/TestRail等の標準項目（リスク: 発生確率×影響×スコア×対応戦略×オーナー×トリガー）を満たすか要現物確認。
  6. ダッシュボードのRAGステータス／ドリルダウン
    PMOダッシュボードの当たり前（RAG信号・変動要因への深掘りリンク・是正記録導線）の充足を要確認。
