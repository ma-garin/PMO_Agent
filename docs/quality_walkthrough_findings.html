この台帳の位置づけ: 先行版 quality_baseline_audit.html はコードのURL経路から机上で作った推定で、横スクロール欠如・日付見切れ・文言崩れといった「描画・挙動レベルの当たり前」を構造的に取りこぼしていた。 本台帳はその欠陥を是正し、全画面を実際に起動・操作して観測した事実のみを記録する。各項目は稼働中アプリで再現確認済み。

★優先度つき重大ギャップ

  1. チケットを開けない／作れない／編集できない 機能欠落
    一覧はあるが行クリックは外部JIRAへ遷移するだけ。アプリ内に詳細・新規作成・編集・状態変更・コメントが一切ない。「＋新規」ボタンも存在せず、外部同期でしか増えない。課題管理ツールの中核が空。
  2. 計画（ガント）の表示・操作品質が破綻 描画/操作
    「全期間」ズームが効かず全体が収まらない／最終タスクが右端で見切れる／横スクロールバーが初期表示で画面外にあり発見不能／右端の日付ラベルが途中で切れる。加えてタスクの追加・編集自体ができない閲覧専用。
  3. LLM(Ollama)呼び出しが常態的に失敗 機能欠損
    管理＞LLM利用状況で失敗率 75.8〜100%。自律運転も「LLMを利用できなかった」とフォールバック表示。モデル名設定だけの問題ではなく、呼び出し自体が通っていない。
  4. 開発者向けの生データ・文言がUIに露出 文言
    自律運転に生Python辞書 {'observed':1,'threshold':1}、空状態に python manage.py init_roadmap_schedule、進捗CSVに英語カラム名、チケット状態バッジに英語 In Progress 等。ユーザーに見せる面に内部表現が漏れている。
  5. 文字列・レイアウトの崩れ（当たり前の描画品質） 描画
    ホーム日付列が「◯/◯ま／で」と単語途中で折返し／進捗見出しが空変数で「の実績推移」と助詞だけ浮く／ナレッジのファイル選択が「選択さ...ません」と切れる／全ページ favicon 404。
